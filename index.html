Positioning

故事从纯视觉导航开始

导航是具身智能落地的基础能力。我们先验证最直接也最困难的一步：不依赖预置地图，让机器人仅凭第一视角视觉和自然语言指令在真实空间中行动。

stage 1

纯视觉方案

输入是任务文本和相机画面，模型直接判断下一步动作。它证明了系统可以从语言到行动闭环，但空间记忆和全局规划能力仍然有限。

stage 2+

逐步补齐空间能力

因此我们继续引入仿真长指令验证、建图定位模块，以及地图与视觉语言模型的结合，让导航从“看见就走”走向“理解空间后再走”。

叙事主线：仿真环境先验证长指令可泛化；真实场景再证明纯视觉具身控制能落地；建图模块补上稳定定位和空间表达；最后的 Luxi-Nav2.0 探索展示地图与模型结合后的语义导航方向。
Roadmap

一条逐步增强的导航路线

每个章节对应一个明确目的：先在仿真里验证长指令路线理解，再上真实机器人证明纯视觉控制落地，然后用建图模块补齐空间底座，最后探索地图与模型融合。

Stage 01仿真长指令验证目的：在可控环境里验证跨房间、跨区域的路线理解。
Stage 02纯视觉真实控制目的：证明机器人只靠语言和视觉也能完成连续任务。
Stage 03建图与定位模块目的：为导航提供稳定地图、位姿和路径规划接口。
Stage 04地图 + 模型探索目的：让模型在地图底座上做语义目标搜索和探索决策。
Evolution: long-instruction simulation → real-world vision-only control → mapping and localization → map-aware vision-language navigation.
Stage 01 · simulation validation

仿真环境：验证长指令和跨区域导航

第一阶段先在 Habitat / Luxi-Nav1.0 仿真环境里验证路线理解。仿真的目的，是在可控环境中系统性测试长指令、跨房间、跨区域导航，再把稳定能力迁移到真实机器人上。

Capability Long-instruction navigation in simulation
in: multi-step natural language route → out: room-to-room navigation, landmark following, target stopping
Stage 02 · vision-only real robot

真实场景：先证明纯视觉方案能跑起来

这一阶段的目的，是验证模型在真实机器人上能把自然语言转成连续动作。第一人称展示机器人“看见什么”，第三人称展示任务是否真正完成。

Capability Long-horizon instruction following in real environments
in: natural language + onboard camera → out: continuous navigation, object search, turning, stopping
Stage 03 · mapping foundation

建图模块：补上长期导航需要的空间底座

纯视觉方案可以启动闭环，但长期、稳定、可复用的导航需要地图和定位。因此我们加入“陆图”建图模块，为机器狗、无人机、轮式机器人等载体提供稠密 3D 地图与实时定位。

Capability Dense mapping, map matching, and robot localization
in: LiDAR / RGB observations → out: PCD map, GLB / OctoMap / OccupancyGrid, robot pose, navigation-ready spatial context

把地图、定位和机器人视角放进同一个演示闭环

这段视频展示陆图系统在 Go2 实机上的模块功能：上方呈现实时构建/加载的空间地图，下方同步展示机器人在真实环境中的运动视角，用来说明地图结果如何服务于后续定位、路径规划和模型决策。

  • 展示点云地图与机器人现场画面的同步关系。
  • 验证建图模块可以为机器人提供可复用的空间表达。
  • 为后续 Luxi-Nav2.0 的“地图 + 模型”语义导航提供底层输入。
01

3D 地图创建

已完成公司环境的雷达点云 PCD 建图，能够形成可用于定位与导航的稠密空间表示。

02

地图格式转换

建图结果可转换为 GLB、OctoMap 体素地图和 SLAM 常用 OccupancyGrid 栅格地图。

03

自动定位匹配

定位库可在 3D 点云地图或 2D 栅格地图中识别机器人位置与朝向，并输出给上层模型。

04

RGB 3D 重建

支持基于 RGB 场景进行三维重建，保留物体相对位置关系和三维形状信息。

Research exploration · map + model

研究探索：把建图结果交给模型做语义决策

最后一阶段面向未来：地图不只是给传统路径规划使用，也可以成为视觉语言模型的空间上下文。Luxi-Nav2.0 展示的是在地图与视觉共同约束下， 模型如何围绕椅子、马桶、床等语义目标选择探索方向，并把“下一步去哪儿看”转化成可执行的导航目标。

Exploration Map-aware vision-language frontier navigation
in: spatial map + target object + visual observation → out: semantic frontier selection, object seeking, navigation target proposal
loop: update explored map → score candidate frontiers with VLM → move toward the most promising area

Takeaway 建图为空间底座，模型做语义决策

Luxi-Nav2.0 方向强调在未知布局中利用地图记录已探索区域和可达边界，再让视觉语言模型根据目标语义判断哪些区域更值得前往。 它不是替代真实机器人导航，而是展示建图模块未来接入模型决策后的研究空间：底层负责稳定定位与路径执行，上层负责语义理解与探索策略。
